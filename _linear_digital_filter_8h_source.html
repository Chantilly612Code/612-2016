16 /**
17  * This class implements a linear, digital filter. All types of FIR and IIR
18  * filters are supported. Static factory methods are provided to create commonly
19  * used types of filters.
20  *
21  * Filters are of the form:
22  * y[n] = (b0*x[n] + b1*x[n-1] + ... + bP*x[n-P) - (a0*y[n-1] + a2*y[n-2] + ... + aQ*y[n-Q])
23  *
24  * Where:
25  * y[n] is the output at time "n"
26  * x[n] is the input at time "n"
27  * y[n-1] is the output from the LAST time step ("n-1")
28  * x[n-1] is the input from the LAST time step ("n-1")
29  * b0...bP are the "feedforward" (FIR) gains
30  * a0...aQ are the "feedback" (IIR) gains
31  * IMPORTANT! Note the "-" sign in front of the feedback term! This is a common
32  * convention in signal processing.
33  *
34  * What can linear filters do? Basically, they can filter, or diminish, the
35  * effects of undesirable input frequencies. High frequencies, or rapid changes,
36  * can be indicative of sensor noise or be otherwise undesirable. A "low pass"
37  * filter smooths out the signal, reducing the impact of these high frequency
38  * components. Likewise, a "high pass" filter gets rid of slow-moving signal
39  * components, letting you detect large changes more easily.
40  *
41  * Example FRC applications of filters:
42  * - Getting rid of noise from an analog sensor input (note: the roboRIO's FPGA
43  * can do this faster in hardware)
44  * - Smoothing out joystick input to prevent the wheels from slipping or the
45  * robot from tipping
46  * - Smoothing motor commands so that unnecessary strain isn't put on
47  * electrical or mechanical components
48  * - If you use clever gains, you can make a PID controller out of this class!
49  *
50  * For more on filters, I highly recommend the following articles:
51  * http://en.wikipedia.org/wiki/Linear_filter
52  * http://en.wikipedia.org/wiki/Iir_filter
53  * http://en.wikipedia.org/wiki/Fir_filter
54  *
55  * Note 1: PIDGet() should be called by the user on a known, regular period.
56  * You can set up a Notifier to do this (look at the WPILib PIDController
57  * class), or do it "inline" with code in a periodic function.
58  *
59  * Note 2: For ALL filters, gains are necessarily a function of frequency. If
60  * you make a filter that works well for you at, say, 100Hz, you will most
61  * definitely need to adjust the gains if you then want to run it at 200Hz!
62  * Combining this with Note 1 - the impetus is on YOU as a developer to make
63  * sure PIDGet() gets called at the desired, constant frequency!
64  */
